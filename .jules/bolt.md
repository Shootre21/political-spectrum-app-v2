## 2024-03-24 - Expensive RegExp Instantiation and Case-conversion in Text Analysis
**Learning:** Instantiating `RegExp` objects inside tight, frequently executed text-analysis loops (e.g., `calculateFramingScore` and `calculateEmotionalScore` in `src/lib/lexicons.ts`) introduces substantial CPU overhead. Furthermore, `.toLowerCase()` on static strings in `.includes()` searches also degrades performance. While the memory noted that `RegExp.prototype.test()` with the `g` flag shouldn't be used due to `lastIndex` carryover, using `String.prototype.match()` with the `g` and `i` flags completely bypasses the `lastIndex` carryover issue while safely counting all overlapping matches.
**Action:** Always pre-compile regular expressions with `g` and `i` flags at the module scope when they are static and used for repeated string scanning. For plain substring searches via `.includes()`, pre-lowercase static array strings to avoid repeated case conversions during iterative evaluations.
