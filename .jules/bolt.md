
## 2025-03-05 - Precompiled Combined Regexes vs Dynamic Iterative Regexes
**Learning:** During text analysis iterations involving loops over lists of static keywords (such as in `src/lib/lexicons.ts`), dynamically instantiating `new RegExp()` repeatedly is a significant performance bottleneck. In local benchmark tests, joining all keywords with `|` (after regex-escaping them) into a single precompiled global `RegExp` per category and then using `text.match(precompiledRegex)` reduced execution time by 60% - 70% compared to the original nested loops and dynamic `match` calls. Additionally, mapping matches back to their original dictionary casing via a reverse-lookup dictionary is much faster than running `.includes()` calls recursively for each word.
**Action:** When searching for any of a known, static list of words inside large text bodies, use a combined precompiled global `RegExp` instead of iterating and building regexes on the fly or running `.includes()` on every term. Use `[...new Set(matches.map(m => m.toLowerCase()))]` and a dictionary if original casing lookup is required. *Update: Ensure that overlapping strings in word lists do not break expected matching behavior before combining with `|`. If distinct overlapping strings must be individually recognized, use a precompiled array of individual RegExp objects instead.*
